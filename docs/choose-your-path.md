# Choose your path

## Start here if you want the combined story

Use `agentpay-wallet-starter` if your goal is to:
- understand how the SDK and MCP server fit together
- run a real paid-tool example first
- adapt the policy or endpoint after first success

## Use Agent Wallet SDK directly if

- you are embedding wallet logic in your own app
- you need library-level control
- you do not need an MCP server surface yet

## Use AgentPay MCP directly if

- you already work in an MCP client flow
- you want tools exposed through MCP quickly
- you already understand the wallet engine layer

## Use this repo first if you are unsure

If you are asking any of these questions, start here:
- Which package owns spend control?
- Which layer should my agent client connect to?
- How do I prove one allowed path and one blocked or approval path before integration?
- Where should I change the endpoint or policy first?

That is the job of this repo.
